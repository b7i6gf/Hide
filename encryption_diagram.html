Encryption & Embedding Flow

How plaintext becomes an encrypted, authenticated bitstream hidden inside an image's least significant bits.

Phase 2 — Key Derivation (encrypted path only)
os.urandom(16)
Random Salt 128 bit
Generated fresh for every single operation. Prevents precomputed rainbow-table attacks. Stored in the payload header — no secret.
Argon2id · RFC 9106 offline profile
Key Derivation
Password + Salt → 256-bit AES key

Phase 3 — Authenticated Encryption
Plaintext assembly
Inner Plaintext
start_marker + plaintext + end_marker

Markers live inside the ciphertext — no structure is visible in raw image data.
→
os.urandom(12)

NIST-recommended 96-bit nonce for GCM. Fresh per operation. Stored in payload header.
→
AES-256-GCM
Encrypt + Authenticate
Key + Nonce + Plaintext → Ciphertext + 128-bit Auth Tag

The auth tag is verified before any decryption on extraction. Tampered or corrupt images are always rejected.

All fields concatenated in order into a single byte sequence. The 4-byte length prefix enables exact extraction — no guessing, no trailing garbage reaching the GCM tag check.

Phase 5 — LSB Embedding (NumPy vectorised)
numpy.unpackbits
Bytes → Bit Array
Payload bytes are unpacked into a flat array of individual bits (MSB first).

np.unpackbits(payload)
→
Vectorised bit masking
Write to Pixel LSBs
Each pixel channel's LSB is replaced with one payload bit.

flat[:n] = (flat[:n] & 0xFE) | bits

Maximum visual change per channel: ±1 — imperceptible.
→
Lossless PNG output
Stego Image
Saved as PNG — lossless compression preserves every LSB exactly. JPEG would destroy the payload.
